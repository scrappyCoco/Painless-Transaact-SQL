Check for column name.

Available context:

  • db - database name
  • schema - schema name
  • table - table name
  • clustered - keywords: clustered/nonclustered

Context variables could be added as named groups (see examples).

Examples:

  • UPPER_CASE: [A-Z]+([_][A-Z]+)*
  • snake_case: [a-z]+([_][a-z]+)*
  • kebab-case: [a-z]+([-][a-z]+)*
  • PascalCase: ([A-Z][a-z]+)+
  • camelCase: [a-z]+([A-Z][a-z]+)+
  • CCS_MyTableName_MyColumnName: (((?<=\bclustered\b.{1,500})C)|((?<=\bnonclustered\b).{0}))CS_\k<table>_([A-Z][a-z]+)+
    Description:
    • (?<=\bclustered\b.{1,500})C) - for clustered index char C must be added
    • ((?<=\bnonclustered\b).{0}) - for nonclustered index prefix must not be added
    • CS_ - constant prefix from ColumnStore
    • \k<table> - table name
    • ([A-Z][a-z]+)+ - name in PascalCase
